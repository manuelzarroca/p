Building my own research survey

Creating a research survey for a concept social app from scratch was a fun project that combined web technologies, translation libraries and spreadsheets.

Project details

Date

October 2025

Tags

Web development, translation

Introduction

In October 2025 I started planning a concept for a new social app that incorporated mental wellbeing features. To see if it would be a good fit for my target audience, I conducted a research experiment with a 20-question survey. The survey was created from scratch and built using HTML, CSS and JavaScript. Additionally, I implemented a specific script to transfer user answers into a custom spreadsheet. Using translater.js, I created over 10 translated versions of the survey to reach even more users.

Creative process

For the creation of this project I used Visual Studio, GitHub and Google Sheets. The project is divided into three simple stages, which are as follows:

Question selection
What did I want to ask and why? What type of answers could return the metrics I was looking for? How could I reach a wider audience? And most importantly, how could I portray my survey as open, friendly and accessible instead of boring, tedious and endless?

Keeping this in mind, I created a 20-question list, which I later organised in a table and classified by number, content, and possible answers. I kept this document handy throughout the whole creative process.


Survey questions organised in table


Survey structuring and assembly
Since this project was quite simple, I merged the design and development stages into one. This was achievable thanks to the computational approach to design I integrate in my work. This stage involved setting a colour palette, choosing a typeface and designing a simple layout, as well as deploying code and configuring a script to transfer answers into my spreadsheet.

This time I picked a basic colour palette, with two variations for light and dark mode, made of white, light grey, a medium grey for secondary text, two shades of dark grey, and blue as the accent colour. My typeface choice was Cascadia Code, a monospaced font that supports almost every major alphabet, including Arabic, Greek and Cyrillic. The font gave my survey a bit of a playful/indie look, while keeping it readable.




Cascadia Code charset diversity


I gave my layout a lot of breathing space to keep things punctual and tidy. By using a slightly darker background and brighter containers for the questions, I created a sense of depth in the page. I also designed custom inputs and text-only checkboxes that were easier to select in contrast to those with small icons.


Breathing space sample in dark mode


Text-only checkboxes in Maori


Using the translater.js library, a simple text button allows users to pick the preferred language in which they would like to answer the survey, and the content translates instantly. To translate the content accurately, I use both translators and dictionaries.


Language selection menu


Within the code, a specific script that detects a form submission event sends all answers to my spreadsheet, from which I can filter by setting diverse parameters.


Spreadsheet with survey results


Testing and publishing
The final stage for this project was testing the functionality of the survey, both visually and on the backend, across devices and viewports. I deployed my code in a specific GitHub repository and made it public using GitHub Pages, then shared the survey with my friends and family, as well as online forums and communities.
